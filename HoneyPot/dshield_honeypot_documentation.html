Initial Setup Process

  1. Hardware Preparation: Assembled Raspberry Pi 5 with CanaKit case and attached SSD storage for improved performance
  2. OS Installation: Downloaded Raspberry Pi Imager from https://www.raspberrypi.com/software/
  3. SD Card Configuration: Flashed Raspberry Pi OS Lite (64-bit) to microSD card using the imager
  4. Pre-boot Configuration: Enabled SSH, set hostname to "pi123", and configured initial credentials

SSH Configuration Steps

  1. SSH Access: Connected to Raspberry Pi using ssh pi@raspberrypi
  2. System Updates: Updated system packages with sudo apt update && sudo apt-get -uy dist-upgrade
  3. Reboot: Restarted system with sudo reboot to apply updates
  4. Git Installation: Installed Git version control with sudo apt-get -y install git

Network Security Implementation

Critical Security Measures:

  • DMZ Isolation: Placed honeypot in DMZ to prevent lateral movement
  • Static IP: Configured DHCP reservation for consistent addressing (192.168.50.84)
  • Port Management: SSH administrative access on port 12222, honeypot on standard ports
  • Firewall Rules: Implemented strict ingress/egress controls

HTTP/HTTPS Web Attack Analysis

The honeypot also captured extensive web-based reconnaissance and attack attempts through HTTP/HTTPS traffic. Analysis of the 404 error logs reveals sophisticated automated scanning and exploitation attempts targeting common web vulnerabilities.

Common Web Attack Patterns Detected:

Router/IoT Exploitation:
  • /cgi-bin/luci/;stok=/locale - OpenWrt router attacks
  • /cgi-bin/login - Router admin panel access
  • /setup.cgi - Device configuration exploitation
Application Vulnerabilities:
  • /SDK/webLanguage - SDK endpoint exploitation
  • /app_dev.php/_profiler/phpinfo - PHP info disclosure
  • /developmentserver/metadatauploader - Dev server attacks
Configuration Files:
  • /.env, /.env.save.1 - Environment files
  • /config.json - Configuration data
  • /config/.env - Laravel environment files
Reconnaissance:
  • /favicon.ico - Service identification
  • /.well-known/security.txt - Security policy discovery
  • /sitemap.xml - Site structure mapping

Web Attack Intelligence Summary:

  • Automated Scanners: Heavy presence of reconnaissance tools (Censys, zgrab) conducting systematic scanning
  • IoT Targeting: Specific focus on router and IoT device vulnerabilities, particularly OpenWrt systems
  • Configuration Hunting: Persistent attempts to access sensitive configuration files and environment variables
  • User Agent Diversity: Mix of legitimate browsers and custom tools indicating both automated and manual reconnaissance
  • Global Sources: Attack traffic originating from diverse geographic locations and IP ranges
  • Multi-Vector Approach: Combination of web application attacks, router exploitation, and information disclosure attempts
